# 测试运行命令

本文档汇总当前工程中与 OpenBPU NoC、Verilator、GPGPU-Sim、Rodinia 相关的常用测试与运行命令。

## 1. Chisel 编译与测试

在仓库根目录执行：

```bash
cd /Users/herbet/code/MyNoC
```

### 1.1 用 sbt 编译工程

```bash
sbt compile
```

### 1.2 运行全部 Chisel 测试

```bash
sbt test
```

### 1.3 运行单个测试

示例：

```bash
sbt "testOnly *VCAllocatorSpec"
sbt "testOnly *RouterArbiterSpec"
```

### 1.4 用 mill 运行测试

如果本机或服务器更适合用 `mill`：

```bash
mill --no-server MyNoC.test
mill --no-server MyNoC.test -z "VCAllocatorSpec"
mill --no-server MyNoC.test -z "RouterArbiterSpec"
```

### 1.5 生成 Verilog / SystemVerilog

```bash
sbt "runMain openbpu.NoCGenerator"
```

如果需要用 `mill`：

```bash
mill --no-server MyNoC.runMain openbpu.NoCGenerator
```

## 2. 构建 Verilator NoC 模型

### 2.1 本地构建 NoC

```bash
./scripts/build_noc.sh
```

这个脚本会完成：

- Chisel 生成 RTL
- 生成 `noc_top.sv` shim
- 调用 Verilator
- 生成 `verilator-noc-lib/build/obj_dir/Vnoc_top__ALL.a`

### 2.2 跳过 Chisel，只重用已生成 RTL

```bash
NOC_SKIP_CHISEL=1 ./scripts/build_noc.sh
```

## 3. 构建 GPGPU-Sim + OpenBPU Wrapper

### 3.1 本地完整构建

```bash
./scripts/build_sim.sh local
```

### 3.2 强制重建 NoC 后再构建模拟器

```bash
NOC_FORCE_BUILD_NOC=1 ./scripts/build_sim.sh local
```

### 3.3 只编 wrapper，不编 GPGPU-Sim

```bash
NOC_SKIP_GPGPUSIM_BUILD=1 ./scripts/build_sim.sh local
```

## 4. Rodinia 运行命令

### 4.1 运行默认 workload

当前脚本默认 `RODINIA_APP=backprop`：

```bash
./scripts/run_rodinia.sh local
```

### 4.2 显式运行 backprop

```bash
RODINIA_APP=backprop ./scripts/run_rodinia.sh local
```

### 4.3 使用 OpenBPU Verilator backend

`network_mode 3` 表示切换到 OpenBPU backend：

```bash
RODINIA_APP=backprop \
GPGPUSIM_NETWORK_MODE=3 \
./scripts/run_rodinia.sh local
```

### 4.4 限周期诊断运行

这个命令适合当前阶段调试 request-network 前进性问题：

```bash
RODINIA_APP=backprop \
GPGPUSIM_NETWORK_MODE=3 \
GPGPUSIM_DEADLOCK_DETECT=0 \
GPGPUSIM_MAX_CYCLE=20000 \
./scripts/run_rodinia.sh local
```

### 4.5 指定其他 workload

例如：

```bash
RODINIA_APP=bfs ./scripts/run_rodinia.sh local
RODINIA_APP=kmeans ./scripts/run_rodinia.sh local
```

注意：

- 当前脚本只显式支持 `backprop`、`bfs`、`kmeans`
- 若新增 workload，需要先补脚本中的路径和默认参数

## 6. 输出结果查看

### 6.1 查看运行日志

脚本默认会把运行日志写到：

```bash
run/backprop_noc_stats.log
```

### 6.2 提取关键 NoC 指标

```bash
grep -E 'Req_Network__packets_(injected|delivered)|Req_Network__avg_latency_cycles|Req_Network__throughput_packets_per_cycle|Reply_Network__packets_(injected|delivered)|Reply_Network__avg_latency_cycles|Reply_Network__throughput_packets_per_cycle|gpgpu_simulation_time|GPGPU-Sim: \*\*\* exit detected \*\*\*' run/backprop_noc_stats.log
```

目前最关键的观测指标包括：

- request 注入包数
- request 送达包数
- request 平均延迟
- request 吞吐
- reply 注入/送达包数
- reply 平均延迟
- 总模拟时间

## 7. 当前推荐调试顺序

如果要继续调试 OpenBPU backend，建议按下面顺序执行：

### 7.1 先验证 Chisel 测试

```bash
sbt test
```

### 7.2 再验证 Verilator 构建

```bash
NOC_FORCE_BUILD_NOC=1 ./scripts/build_sim.sh local
```

### 7.3 最后跑 bounded Rodinia

```bash
RODINIA_APP=backprop \
GPGPUSIM_NETWORK_MODE=3 \
GPGPUSIM_DEADLOCK_DETECT=0 \
GPGPUSIM_MAX_CYCLE=20000 \
./scripts/run_rodinia.sh local
```

## 8. 说明

当前工程已经可以做到：

- OpenBPU NoC 经 Verilator 接入 GPGPU-Sim
- 在服务器上运行 Rodinia workload
- 打印 NoC latency / hops / throughput

但当前 request path 仍存在明显前进性问题，因此研究实验建议优先使用限周期诊断命令收集统计，再继续修 request-network 的送达率。
